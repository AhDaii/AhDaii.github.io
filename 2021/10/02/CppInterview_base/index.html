c++面试知识点_语言基础

0xygen2021-10-022023-09-21

语言基础

  1. 简述下C++语言的特点：

    1. 在c语言的基础上多了面向对象的机制，兼容c语言
    2. 三大特性：封装、继承、多态
    3. 可读性好，易于扩充
    4. 运行效率高
    5. 相比c语言更加安全，增加了const、引用、四种cast转换（static_cast, dynamic_cast, const_cast, reinterpret_cast）、智能指针(auto_ptr, shared_ptr, unique_ptr, weak_ptr)
    6. 可复用性高，具有模版

  2. 说说c语言和c++的区别

    1. c++兼容c语言，但c++拥有许多新特性，如引用、智能指针、auto、const等
    2. c++是面向对象的，c语言是面向过程的
    3. c语言不太安全，例如内存泄漏、强制转换的不确定性等。而c++通过新的特性改善了这些问题，例如通过四种cast转换解决强制转换的问题、通过智能指针解决内存泄漏的问题，以及使用try-catch来捕获异常等等
    4. 可复用高，根据模版技术衍生的标准模版库STL相比c语言的函数库更加灵活、通用。

  3. c++中struct和class的区别

    1. struct一般用来描述一个数据结构集合，而class通常是对一个对象的封装。
    2. struct中默认访问控制权限是public，而class默认是private
    3. 在继承中，struct默认是公有继承，而class是私有继承
    4. class关键字还可以定义模版参数，而struct不行

  4. include文件顺序以及””和<>的区别

    1. <>内的文件是系统文件，查找路径：编译器设置的头文件路径->系统变量
    2. “”内的文件是用户自定义文件，查找路径：当前头文件的目录->编译器设置的头文件路径->系统变量

  5. c++中结构体和c语言结构体的区别

    1. c的结构体不允许函数存在，但是c++允许，而且该函数可以是虚函数
    2. c结构体的对于内部成员的访问权限只能是public，而c++允许protect、private、public
    3. c的结构体不能继承，c++的结构体支持继承
    4. c中使用结构体必须带struct关键字，c++可以省略

  6. 导入c函数的关键字是什么，c++编译和c有什么不同

    1. 关键字：**extern “C”**，加上该关键字可以使得c++正确调用c的函数，这部分会按照c语言的标准进行编译
    2. 区别：因为c++支持重载，所以编译器在编译c++中的函数时会将函数的参数类型也会包括在函数的名称中，而不是只有函数名，而c语言编译函数时只有函数名

  7. c++从代码到可执行二进制文件的过程：预处理->编译->汇编->链接

  8. 说说static关键字的作用

    1. 定义全局静态变量和局部静态变量：初始化的静态变量会在数据段分配内存，未初始化的静态变量会在BSS段分配内存。
    2. 定义静态函数和静态变量：静态函数和静态变量只能在本源文件中使用
    3. 定义静态成员变量和静态成员函数：可以被当成全局变量那样使用，但被隐藏在类的内部静态成员变量和静态成员函数都只有一块单独的存储区，所有对象都共享这一段静态存储空间。注意！静态成员函数中没有this指针！因为静态成员函数不属于任何一个对象！

  9. 说说指针和数组的区别

    1. 概念：
      • 数组适用于存储多个相同类型的集合。数组名是首元素的地址
      • 指针相当于一个变量，指针中存储的是其他变量在内存中的地址。指针名指向了内存的首地址。
    2. 区别：
      1. 赋值：同类型的指针变量可以相互赋值；数组不行，只能一个个元素赋值
      2. 存储方式：数组在内存中是一块连续的内存空间，不是存储在栈上就是在静态区；而指针存储空间不好确定，因为其本身就是变量，而且存放的也是变量
      3. 大小：数组的大小为sizeof(数组名)；而指针的大小一般在32位平台上为4B，在64位平台上位8B

  10. 说说什么是函数指针，如何定义函数指针，有什么使用场景

    • 函数指针就是只指向函数的指针。每个函数都有一个入口地址，这个入口地址就是函数指针所指向的地址。

  11. 说说静态语言什么时候初始化？

    • 对于c的全局或静态变量，初始化发生在任何代码执行之前，也就是编译期初始化

    • c++规定，全局或静态对象，当且仅当对象首次用到时才进行构造，也就是运行期初始化

  12. nullptr调用成员函数可以吗？为什么？

    • 可以，因为在编译期间对象就绑定了函数的地址，与指针是否为空无关，但是如果涉及到关于this的解引用操作，并且通过*this去操作成员变量或者函数中涉及成员变量都会报错

  13. 说说什么是野指针，怎么产生的，如何避免？

    1. 概念: 野指针就是指针指向的位置是不可知的(随机的、不正确的、没有明确限制的)
    2. 产生原因: 释放内存后没有及时将指针置空，仍然指向该内存，那么可能出现非法访问的错误。
    3. 避免办法
      1. 初始化时置为NULL
      2. 申请内存后判断是否为空
      3. 指针释放后置为NULL
      4. 使用智能指针

  14. 说说静态局部变量、全局变量、局部变量的特点，以及使用场景

    1. 作用域：c++里的作用域可以分为6种：全局、局部、类、语句、命名空间、文件作用域
      • 全局变量：全局作用域，可以通过extern作用于其他的未定义的文件中
      • 静态全局变量：全局作用域+文件作用域，无法在其他文件中使用
      • 局部变量：局部作用域，如函数参数，函数内的局部变量等
      • 静态局部变量：局部作用域，只会被初始化一次
    2. 所在内存空间：除了局部变量在栈上，其他的都在静态存储区中
    3. 生命周期：因为局部变量在栈中，超过其作用域就会被回收内存；其他存储在静态存储区中的变量只有在程序结束时才会回收内存

  15. 说说宏函数和内联函数的区别

    • 宏函数本质不是函数，只是使用起来像函数，预处理器用替换代码的方式替代函数的调用，省去了压栈和退栈的过程，效率更高；而内联函数本质就是函数，内联函数一般用于调用时间比执行时间更长的函数，内部一般只包含简单的函数，不能包含复杂的语句，而且内联函数本身不能调用自身。
    • 宏函数在预编译的时候把所有宏名称进行替换的，也就是字符串替换；而内联函数是通过编译时进行代码插入的，编译器会在调用内联函数的地方直接把内联函数的内容展开，这样可以省去函数调用的开销，提高效率。
    • 宏定义没有参数检查，无论对错都是直接替换的；而内联函数在编译的时候会进行参数检查，内联函数同时满足函数的性质，如参数列表、返回值等

  16. 说说i++和++i的区别

    1. 赋值的顺序不同：++i是先加再赋值，而i++是先赋值再加，++i和i++都是分为两步完成的
    2. 效率不同：++i的执行相比i++更快，因为少了个使用临时变量保存原值的操作。

  17. 说说new和malloc的区别，各自底层实现的原理

    1. new是操作符，malloc是个函数
    2. new在调用的时候先调用malloc分配内存，然后调用构造函数，在释放的时候先调用析构函数，然后使用free回收内存；malloc只会申请内存
    3. malloc需要指定申请内存的大小，而且返回的指针需要进行强制转换；而new不需要指定内存大小，返回的指针也不需要强转。
    4. new可以被重载；malloc不行
    5. new分配内存更直接和安全
    6. new发生错误抛出异常，malloc则返回NULL

    malloc底层实现:当开辟的空间小于 128K 时，调用 brk()函数；当开辟的空间大于 128K 时，调用mmap()。malloc采用的是内存池的管理方式，以减少内存碎片。先申请大块内存作为堆区，然后将堆区分为多个内存块。当用户申请内存时，直接从堆区分配一块合适的空闲快。采用隐式链表将所有空闲块，每一个空闲块记录了一个未分配的、连续的内存地址。

    new底层实现:关键字new在调用构造函数的时候实际上进行了如下的几个步骤：

    1. 创建一个新的对象
    2. 将构造函数的作用域赋值给这个新的对象（因此this指向了这个新的对象）
    3. 执行构造函数中的代码（为这个新对象添加属性）
    4. 返回新对象

  18. 说说const和define的区别

    • const一般用于定义常量，而define用于定义宏，define也可以用于定义常量。当它们都定义常量时，有以下区别：
      1. const生效于编译期，而define生效于预处理阶段
      2. const定义的常量，是存储在内存中的，需要额外的空间，而define定义的常量，运行时是直接的操作数，并不会放到内存中
      3. const定义的常量是带类型的，会进行类型的检查，而define定义的常量是无类型的，不会进行类型的检查

  19. 说说c++中函数指针和指针函数的区别

    1. 定义不同：

      • 指针函数本质是一个函数，表示其返回值是个指针
      • 函数指针本事是一个指针，其指向一个函数

  20. 说说const int *a, int const *a, cosnt int a, int *const a, const int* const a分别是什么，有什么特点

    • const int *a: 表示a指向的变量不可以被修改，a是个常量指针
    • int const *a: 同上
    • const int a: 表示a是个常量，不能被修改
    • int *const a: 表示a指向的变量可以被修改，但是a指针本身无法被修改，不能指向其他变量，a是个常指针
    • const int* const a：表示a指向的变量和其自身都无法被修改，a是个常量的常指针

  21. 说说使用指针需要注意什么？

    1. 定义指针的时候，先初始化为NULL
    2. 使用malloc或者new申请内存后，应当立即检查指针是否为NULL，防止指针使用值为NULL的内存
    3. 避免数字或指针的下标越界
    4. 申请和释放必须配对，防止内存泄漏
    5. 使用free或者delte之后，应当立即将指针设置为NULL，防止“野指针”

  22. 说说内联函数和函数的区别，内联函数的作用

    • 区别：
      1. 内联函数比普通函数多了inline关键字
      2. 内联函数避免了函数调用的开销，普通函数有函数调用的开销
      3. 普通函数被调用的时候，需要进行**寻址(函数入口地址)**，内联函数不需要寻址
      4. 内联函数要求代码简单，不能包含复杂的语句；普通函数没有这个要求
    • 内联函数的作用：内联函数在调用的时候，是将调用处利用函数体进行替换的，避免了调用函数的开销

  23. 简述c++有几种传值的方式，之间的区别是什么？

    1. 值传递：形参即使在函数体中发生变化，也不会影响实参的值。若通过值传递传递对象，会发生对象的拷贝行为
    2. 引用传递：形参在函数体中发生变化，会影响实参的值。但用引用传递对象，不会发生对象的拷贝行为，效率更高
    3. 指针传递(地址传递)：在指针指向不发生改变的前提下，指针指向的内容在函数体中发生变化，实参也会发生变化
